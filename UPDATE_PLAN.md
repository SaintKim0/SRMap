# TasteMap 앱 업데이트 계획 (Roadmap)

본 문서는 TasteMap의 단기 및 중기 업데이트 계획을 정리합니다. 사용자 피드백과 프로젝트 목표에 따라 상시 업데이트됩니다.

---

## 🚀 1단계: 데이터 고도화 및 UI 마감 (진행 중 - v1.1.5)
목표: 데이터 신뢰성 확보 및 시각적 완성도 통일

### [완료] 데이터 강화
- **미슐랭 가이드 2025 보완**: 전국의 미슐랭 2025 레스토랑 데이터 추가 및 네이버 API 기반 좌표/상세 정보 보정.
- **셰프 정보 통합**: 미슐랭 2025 레스토랑의 대표 셰프 성함 수집 및 정제 완료 (140여 건의 오탐 제거).
- **신규 콘텐츠**: '전현무계획 3' 맛집 데이터 통합 완료.

### [완료] UI/UX 개선
- **카드 디자인 통일**: 미슐랭 카드에 셰프 이름을 노출하고, 모든 맛집 카드의 높이를 '흑백요리사' 카드와 동일하게 통일.
- **동적 썸네일 시스템**: '전현무계획3' 등 긴 제목의 프로그램을 위한 자동 줄바꿈 및 타이포그래피 최적화.

---

## 🎨 2단계: 이미지 최적화 및 검색 기능 강화 (v1.2.0)
목표: 풍부한 시각 자료 제공 및 원하는 장소로의 빠른 접근

### 시각적 경험 강화
- **GitHub 이미지 호스팅 최적화**: 수집된 수만 장의 이미지를 안정적으로 제공하기 위한 GitHub LFS(Large File Storage) 설정.
- **이미지 로딩 성능 개선**: WebP 변환 및 저사양 기기 최적화.

### 검색 및 필터 고도화
- **지역 기반 검색**: 시/군/구 단위의 직관적인 지역 필터 UI 구현.
- **상세 필터**: 업종별(한식/일식/카페 등), 미슐랭 티어별 정교한 필터링.
- **지도 탐색 개선**: 클러스터링(Clustering)을 통한 밀집 지역 지도 로딩 속도 향상.

---

## 🤝 3단계: 커뮤니티 및 개인화 서비스 (v2.0.0)
목표: 사용자 참여 유도 및 자신만의 맛집 리스트 공유

### 개인화 고도화
- **사용자 계정 통합**: Firebase/Supabase 연동을 통한 기기 간 데이터 동기화 (찜/방문 목록).
- **미식 리포트**: 사용자가 방문한 셰프/프로그램 기반의 개인 맞춤형 성향 분석 확장.

### 소셜 및 공유
- **리뷰 시스템**: 별점이 아닌 텍스트 기반의 생생한 한줄평 기능.
- **맛집 코스 공유**: "나만 알고 싶은 미식 코스"를 생성하고 친구에게 링크로 공유하는 기능.
- **공공 API 연동**: 날씨나 요일에 따른 실시간 맛집 추천.

---

## 🛠 품질 관리 및 성능
- **지속적 데이터 클렌징**: 네이버/카카오 API 정기 대조를 통한 폐업 정보 반영.
- **배터리 최적화**: 위치 서비스 및 지도 이미지 캐싱 로직 최적화.
