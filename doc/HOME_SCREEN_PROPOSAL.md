# 홈 화면 구성 제안 (맛집지도)

**적용 완료 (2025.01.31)**  
- 앱 안내 카드 추가(접기/펼치기), SectorCard·작품 카드·KMDb 카드 스타일 통일(borderRadius 16, boxShadow), 섹션 간격·패딩 유지.

---

## 1. 현재 상태 요약

- **상단**: AppBar(맛집지도 + 검색 아이콘)
- **탭**: 전체보기 | 흑백요리사 | 미슐렝 | 예능 (화면 폭 80% 균등)
- **본문(전체보기)**: "맛집 둘러보기" → SectorCard 3개(흑백요리사 / 미슐렝 코리아 / 예능) → "최근 추가" 리스트
- **다른 페이지**: 마이페이지는 Card + padding 16 + borderRadius 16 + boxShadow 패턴 사용

---

## 2. 홈 화면 구성 제안 (세로 순서)

| 순서 | 섹션 | 설명 |
|------|------|------|
| 1 | **앱 안내** | 앱이 뭔지, 뭘 할 수 있는지 짧은 설명 (접을 수 있으면 더 좋음) |
| 2 | **카테고리 탭** | 기존처럼 전체보기 / 흑백요리사 / 미슐렝 / 예능 (유지) |
| 3 | **맛집 둘러보기** | 흑백요리사·미슐렝·예능 SectorCard (유지) |
| 4 | **최근 추가** | 최근 추가된 맛집 리스트 (유지) |

→ **앱 안내**만 맨 위에 추가하고, 나머지 흐름은 그대로 두는 구성을 권장합니다.

---

## 3. 앱 안내 섹션 제안

### 3.1 넣을 내용 (문구 예시)

- **한 줄 소개**  
  `"흑백요리사, 미슐렝, 예능에 나온 맛집을 한곳에서."`

- **세 가지 포인트 (선택)**  
  - 프로그램·가이드별로 맛집 보기  
  - 지도에서 위치 확인 후 길찾기  
  - 저장·방문 기록으로 나만의 맛집 리스트  

- **행동 유도 (선택)**  
  - "맛집 둘러보기" 또는 "지도에서 보기" 같은 짧은 CTA 버튼

### 3.2 UI 형태 제안

- **카드형**  
  마이페이지의 통계/뱃지 카드와 동일한 스타일  
  - `padding: 16`, `borderRadius: 16`, `boxShadow`, `Theme.cardColor` / `cardBackgroundColor`  
  - 제목(예: "맛집지도란?") + 본문 텍스트 + (선택) CTA 버튼  

- **접기/펼치기**  
  - 기본: 한 줄 소개만 보이기  
  - 탭 시: 세 가지 포인트 + CTA 펼치기  
  → 첫 방문자는 설명을 보고, 단골은 접어두고 스크롤 부담을 줄이기 위함  

이렇게 하면 **앱 설명**은 확실히 들어가면서, **다른 페이지와 같은 카드 스타일**로 통일할 수 있습니다.

---

## 4. UI 통일 제안

### 4.1 공통으로 맞출 것

| 항목 | 제안 |
|------|------|
| **섹션 카드** | 홈·마이·검색 등에서 "카드형 섹션"은 동일 스타일: `borderRadius: 16`, `padding: 16`, `Theme.cardColor` / `cardBackgroundColor`, `boxShadow` (blur 10, offset 0,4) |
| **섹션 제목** | 동일한 스타일: `titleMedium` + bold, 필요 시 아이콘은 `Theme.primaryColor` |
| **여백** | 세로 섹션 간격 `16` 또는 `24`로 통일 (홈·마이·검색 동일) |
| **AppBar** | 높이·타이틀 스타일 유지, 필요 시 마이페이지도 "마이" 등 짧은 타이틀로 통일 |

### 4.2 홈 전용 조정

- **앱 안내 카드**  
  - 마이페이지의 "저장/방문/리뷰" 카드와 같은 Container/Card 스타일 사용  
- **"맛집 둘러보기" 블록**  
  - 지금도 카드 형태이므로, 배경색·모서리·그림자를 `AppTheme.cardBackgroundColor`, `borderRadius: 12~16`, `boxShadow` 한 종류로 맞추기  
- **빈 상태(EmptyState)**  
  - 문구·아이콘 스타일을 검색/리스트 화면과 동일하게 두면 일관성 있음  

---

## 5. 구현 시 우선순위 제안

1. **앱 안내 카드 추가**  
   - 맨 위에 "맛집지도란?" 카드 추가  
   - 한 줄 소개 + (선택) 접기/펼치기 + 상세 3줄 + CTA  
   - 스타일: 마이페이지 카드와 동일 (padding 16, borderRadius 16, boxShadow)

2. **카드 스타일 통일**  
   - 홈의 SectorCard·작품 카드 등에 `Theme.cardBackgroundColor`, `borderRadius`, `boxShadow` 일괄 적용  
   - 마이페이지와 동일한 수치 사용

3. **섹션 간격·타이포 통일**  
   - 홈 본문 padding/세로 간격을 16·24 기준으로 정리  
   - 섹션 제목은 `_buildSectionHeader` 또는 공통 위젯 하나로 통일

4. **(선택) 공통 위젯 분리**  
   - `SectionCard`(제목+내용+선택 CTA), `SectionHeader`를 공통 위젯으로 두고 홈/마이/검색에서 재사용하면 유지보수와 통일성이 좋아짐  

---

## 6. 정리

- **구성**: 앱 안내(맨 위) → 카테고리 탭 → 맛집 둘러보기 → 최근 추가  
- **앱 안내**: 카드형, 한 줄 소개 필수, 상세·CTA는 선택, 접기/펼치기 권장  
- **UI 통일**: 카드 스타일·섹션 제목·여백을 마이페이지와 맞추고, 필요 시 공통 위젯으로 빼기  

이 순서로 적용하면 "앱 설명이 있는 홈"과 "다른 페이지와 통일된 UI"를 동시에 만족할 수 있습니다.
