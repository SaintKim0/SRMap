# Stage 8: UI/UX 폴리싱 PRD

## 1. 개요
### 목표
앱의 시각적 완성도와 사용성을 높이기 위해 디자인 디테일을 다듬고, 자연스러운 애니메이션을 추가합니다. '프리미엄'하고 '매끄러운' 사용자 경험을 제공하는 것이 목표입니다.

### 범위
- **애니메이션**: Hero 애니메이션 (이미지 전환), 리스트 아이템 등장 애니메이션.
- **디자인 개선**: `LocationCard` 디자인 고도화 (그림자, 여백, 타이포그래피).
- **피드백 강화**: 로딩 스켈레톤, 에러/빈 화면 디자인 개선, 버튼 터치 피드백.

---

## 2. 주요 개선 사항

### 2.1 Hero 애니메이션 (Image Transition)
- **적용 대상**: `LocationCard` (홈, 검색, 북마크) -> `LocationDetailScreen`의 상단 이미지.
- **효과**: 리스트의 작은 썸네일이 상세 화면의 큰 헤더 이미지로 부드럽게 확장됨.
- **구현**: `Hero` 위젯 사용. `tag`는 `location.id`를 사용하여 유니크하게 관리.

### 2.2 리스트 등장 애니메이션 (List Entrance)
- **적용 대상**: 검색 결과, 찜 목록.
- **효과**: 화면 진입 시 아이템들이 순차적으로 아래에서 위로 페이드인되며 나타남 (Staggered Animation).
- **구현**: `flutter_animate` 패키지나 커스텀 `AnimationController` 사용 (패키지 추가 없이 커스텀 구현 지향).

### 2.3 카드 디자인 (LocationCard)
- **개선점**:
    - 썸네일 비율 고정 (4:3 또는 16:9) 및 `BoxFit.cover` 적용.
    - 텍스트 가독성 향상 (제목 굵게, 주소/카테고리 색상 조정).
    - 터치 영역 명확화 및 `InkWell` 리풀 효과 적용.

### 2.4 로딩 및 에러 상태 (Loading & Error)
- **로딩**: 단순 `CircularProgressIndicator` 대신 컨텐츠 윤곽을 보여주는 스켈레톤(Skeleton) UI 적용 (커스텀 위젯 코딩).
- **빈 화면**: `EmptyState` 위젯 아이콘 크기 키우고, 친절한 문구("아직 찜한 장소가 없어요. 마음에 드는 곳을 찾아보세요!")로 변경.

---

## 3. 기술 구현 계획

### 3.1 UI Components
- **`LocationCard` 수정**: `Hero` 위젯 감싸기.
- **`LocationDetailScreen` 수정**: 상단 `PageView`/`Image`를 `Hero`로 감싸기.
- **`FadeInUp` 위젯 생성**: 자식 위젯에 등장 애니메이션을 적용하는 래퍼 위젯.

### 3.2 작업 순서
1.  **Hero 적용**: 카드와 상세 화면 연결.
2.  **UI 디테일 수정**: 카드 디자인, 패딩, 폰트 스타일 조정.
3.  **애니메이션 추가**: 리스트 등장 효과.
4.  **예외 처리 디자인**: 빈 화면/로딩 화면 개선.

## 4. 테스트 시나리오

- [ ] 검색 결과가 나올 때 리스트가 부드럽게 나타나는지 확인.
- [ ] 이미지가 로딩 중일 때 회색 박스(Placeholder)가 적절히 보이는지 확인.
