# SceneMap 개발 로드맵

## 프로젝트 개요
드라마/영화 촬영지 정보를 제공하는 모바일 앱으로, K-콘텐츠 팬들의 '세트 제팅(Set-jetting)' 경험을 지원합니다.

## 개발 단계

### Phase 1: MVP (Minimum Viable Product) - 2개월
**목표**: 기본적인 촬영지 정보 제공 및 지도 기능

#### Stage 1: 기획 및 문서화 (1주)
- PRD 작성 및 화면 설계
- 데이터 모델 설계
- 기술 스택 확정

#### Stage 2: 데이터 모델 구현 (1주)
- Location, Content, Scene, User 모델 생성
- JSON 직렬화 설정

#### Stage 3: 로컬 데이터 저장소 (1주)
- SQLite 데이터베이스 구축
- Repository 패턴 구현
- 샘플 데이터 준비

#### Stage 4: UI 화면 구현 (3주)
- 홈 화면
- 촬영지 목록 및 상세 화면
- 검색 기능
- 기본 네비게이션

#### Stage 5: 지도 기능 (2주)
- 지도 연동 (Google Maps/Naver Maps)
- 위치 기반 서비스
- 경로 안내

**MVP 출시 목표**: 특정 인기 드라마 1-2개의 촬영지 정보 제공

---

### Phase 2: 차별화 기능 추가 - 2개월
**목표**: 코스 추천 및 게이미피케이션으로 사용자 참여 유도

#### Stage 6: 코스 추천 기능 (2주)
- 테마별 코스 설계
- 동선 최적화
- 코스 상세 정보

#### Stage 7: 스탬프 투어 (3주)
- 위치 기반 체크인
- 스탬프 수집 시스템
- 보상 및 배지 시스템

#### Stage 8: AR 오버레이 (3주)
- AR 라이브러리 연동
- 장면 오버레이 기능
- 촬영 가이드

**Phase 2 목표**: 사용자 재방문율 향상 및 체류 시간 증가

---

### Phase 3: 커뮤니티 및 확장 - 2개월
**목표**: 사용자 생성 콘텐츠 및 커뮤니티 활성화

#### Stage 9: 커뮤니티 기능 (3주)
- 리뷰 시스템
- 사진 공유
- 실시간 정보 업데이트 (크라우드소싱)

#### Stage 10: 사용자 인증 (1주)
- 소셜 로그인
- 프로필 관리

#### Stage 11: 백엔드 연동 (4주)
- API 서버 구축 또는 Firebase 연동
- 데이터 동기화
- 이미지 저장소

**Phase 3 목표**: 사용자 커뮤니티 형성 및 데이터 품질 향상

---

### Phase 4: 완성도 및 배포 - 1개월
**목표**: 앱 스토어 출시 준비

#### Stage 12: UI/UX 개선 (2주)
- 애니메이션 및 인터랙션
- 다국어 지원
- 접근성 개선

#### Stage 13: 테스트 (1주)
- 단위/통합 테스트
- 사용자 테스트

#### Stage 14: 배포 준비 (1주)
- 앱 아이콘 및 스크린샷
- 스토어 등록
- 법적 문서 준비

**Phase 4 목표**: Google Play Store 및 App Store 출시

---

### Phase 5: 운영 및 성장 - 지속적
**목표**: 사용자 피드백 기반 개선 및 콘텐츠 확장

#### Stage 15: 운영 및 개선
- 사용자 피드백 수집 및 반영
- 버그 수정 및 성능 최적화
- 새로운 드라마/영화 촬영지 추가
- 지자체 협업 및 제휴
- 수익 모델 실험 (광고, 프리미엄 기능 등)

## 기술 스택

### Frontend (Flutter)
- **상태 관리**: Provider
- **라우팅**: GoRouter
- **지도**: Google Maps / Naver Maps
- **AR**: ARCore (Android) / ARKit (iOS)
- **로컬 저장소**: SQLite, SharedPreferences
- **이미지**: CachedNetworkImage, ImagePicker

### Backend (선택사항)
- **Option 1**: Firebase (빠른 개발)
  - Firestore (데이터베이스)
  - Firebase Storage (이미지)
  - Firebase Auth (인증)
- **Option 2**: Custom API Server
  - Node.js / Python (FastAPI)
  - PostgreSQL / MongoDB
  - AWS S3 (이미지)

### 기타
- **분석**: Firebase Analytics, Google Analytics
- **푸시 알림**: Firebase Cloud Messaging
- **CI/CD**: GitHub Actions, Codemagic

## 리스크 및 대응 전략

### 1. 저작권 문제
**리스크**: 드라마/영화 스틸컷 사용 시 저작권 침해
**대응**: 
- 공식 제공 이미지 사용 또는 라이선스 획득
- 일러스트레이션으로 대체
- 외부 링크 연결 (공식 사이트, YouTube 등)

### 2. 데이터 유지보수
**리스크**: 촬영지 폐업, 리모델링 등으로 정보 변경
**대응**:
- 사용자 제보 시스템 구축
- 주기적인 데이터 검증
- 지자체/관광공사와 협업

### 3. 수익 모델
**리스크**: 초기 수익 창출 어려움
**대응**:
- 주변 맛집/숙소 제휴 광고
- 프리미엄 오디오 가이드 판매
- 지자체 관광 활성화 지원 사업 참여
