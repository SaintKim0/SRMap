# 이미지 호스팅 방법 비교 및 추천

## 🎯 추천 순위

### 1위: **GitHub (raw.githubusercontent.com)** ⭐⭐⭐⭐⭐
**가장 추천하는 방법**

#### 장점
- ✅ **완전 무료** (무료 플랜으로 충분)
- ✅ **설정 간단** (이미 GitHub 사용 중)
- ✅ **CDN 효과** (GitHub의 글로벌 CDN)
- ✅ **버전 관리** (Git으로 이미지 관리)
- ✅ **안정성** (GitHub 인프라)
- ✅ **별도 서버 불필요**

#### 단점
- ⚠️ 저장소당 1GB 권장 (무료 플랜)
- ⚠️ 단일 파일 100MB 제한
- ⚠️ 대용량은 LFS 필요 (무료 1GB)

#### 적합한 경우
- ✅ **현재 프로젝트에 최적!**
- ✅ 이미지 약 15,000개 × 평균 3개 = 45,000개
- ✅ 평균 이미지 크기 200KB 가정 → 약 9GB 필요
- ✅ **GitHub LFS 사용 시 무료 1GB + 유료 $5/월로 50GB**

#### 비용
- 무료: 1GB 저장소 + 1GB 대역폭/월
- 유료: $5/월 (50GB 저장소 + 50GB 대역폭/월)

- ✅ 빠른 설정 원함

#### 비용
- 무료: 100GB 대역폭/월
- 유료: $20/월 (더 많은 용량)

## 💡 최종 추천: **GitHub + LFS**

### 이유

1. **현재 상황에 최적**
   - 이미 GitHub 사용 중
   - 설정이 간단
   - 무료로 시작 가능

2. **용량 계산**
   - 약 15,000개 장소
   - 평균 3개 이미지/장소
   - 평균 200KB/이미지
   - **총 약 9GB 필요**

3. **비용 효율**
   - 무료: 1GB (테스트용)
   - 유료: $5/월 (50GB) ← **추천!**
   - AWS 대비 저렴하고 예측 가능

4. **간편함**
   - Git으로 관리
   - 자동 배포
   - 버전 관리

### 실행 계획

#### 단계 1: 무료로 시작 (1GB)
```bash
# 1. 이미지 다운로드 (일부만)
python scripts/download_images.py

# 2. GitHub에 업로드
python scripts/upload_to_github.py
```

#### 단계 2: 필요시 LFS 유료 플랜 ($5/월)
- 이미지가 많아지면
- GitHub LFS 유료 플랜으로 업그레이드
- 50GB까지 사용 가능

## 🚀 추천 실행 순서

### 즉시 실행 (GitHub 무료)

1. **GitHub Personal Access Token 생성**
   ```
   Settings → Developer settings → Personal access tokens
   ```

2. **.env 파일 설정**
   ```env
   GITHUB_TOKEN=ghp_xxx
   GITHUB_REPO=username/repo-name
   IMAGE_SERVER_BASE_URL=https://raw.githubusercontent.com/username/repo-name/main/assets/images
   ```

3. **이미지 다운로드 및 업로드**
   ```bash
   python scripts/download_images.py
   python scripts/upload_to_github.py
   ```

4. **앱에서 테스트**
   - 앱 재실행
   - 이미지가 GitHub에서 로드되는지 확인

### 필요시 업그레이드

- 이미지가 1GB 초과하면
- GitHub LFS 유료 플랜 ($5/월) 구독
- 50GB까지 사용 가능
